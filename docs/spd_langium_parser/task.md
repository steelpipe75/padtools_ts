# タスク概要
SPDというDSLのパーサーを、現在の正規表現ベースの実装からLangiumを用いた実装に書き直す。

## 現在の実装
- `src/spd/parser.ts` にて独自実装されている。
- タブ文字によるインデントでネスト（階層）を表現している。
- `:` から始まる行はコマンド（if, while, callなど）として扱われる。
- それ以外の行はProcessNodeとして扱われる。

## ゴール
- Langiumを用いてSPDの文法を定義する。
- パース結果が現在の `Node` / `NodeListNode` 等の既存ASTと同じデータ構造になるように変換処理を実装する。
- 既存のテストが通り、挙動が変化しないようにする。
