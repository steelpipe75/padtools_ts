# タスク: SPD DSLパーサーのLangiumへの移行

## 背景と目的
本プロジェクトは、SPD（Structured Programming Diagram）というDSLを入力とし、アルゴリズムを図示したPAD（Problem Analysis Diagram）記法のSVG画像を生成するツールです。
現状、SPDのパースは正規表現ベースで行われていますが、より堅牢で拡張性の高い構文解析を実現するため、Langiumを用いたパーサーに書き換えることが求められました。

## 要件
1. `src/spd/` 以下にある既存の正規表現パーサーを、Langiumを使用した実装に移行する。
2. AST構築の全体的な構造は維持しつつ、各行の構文解析（文の種類判定や引数抽出など）をLangiumで行うアプローチ（アプローチA）を採用する。
3. 移行作業は専用のブランチを作成し、そこにコミットする。
