# タスク: Langiumによるフルパーサへの移行 (アプローチB)

## 背景と目的
現在、SPD(Structured Pad Diagram)からPADのSVG画像を生成するプロジェクトを開発しています。
従来のSPDパーサは、正規表現ベースで「行単位の解析」を行っており、インデントの深さを手動でトラッキングしながら木構造(AST)を構築していました。
これをLangiumを用いた本格的な言語プロセッサに移行する作業を行いました。最初のアプローチ(アプローチA)では「行単位の解析」を維持したままコマンド解釈部分のみLangium化しましたが、今回はこれをさらに発展させ、Langiumの機能をフル活用してインデントベースのツリー構造全体をLangiumで直接パースする「フルパーサ (アプローチB)」への完全移行を目標としました。

## 実装内容
1. **LangiumカスタムLexerの導入**: Langiumはデフォルトでインデント（Pythonライクなオフサイドルール）をサポートしていないため、Chevrotainのトークンストリームを加工し、インデントレベルに応じて`INDENT` / `DEDENT` トークンを動的に挿入する `SpdLexer` を実装しました。
2. **Langiumグラマーのフル構造化**: `spd.langium` において、`Block` ルールを定義し、`INDENT statements+=Statement+ DEDENT` のように木構造を直接パースできるようにグラマーを修正しました。
3. **AST変換処理のリファクタリング**: `parser.ts` を全面的に書き換え、Langiumが生成したツリー構造（AST）を直接走査し、従来の `Node` 構造（`ProcessNode`, `IfNode`, `LoopNode` など）に変換するロジックを実装しました。
4. **互換性の確保と例外処理**: 古い正規表現パーサのコーナーケース（マルチライン `@`、インデントの異常、コマンド引数の有無チェック）に対する例外スローの挙動を、LangiumのエラーやAST情報を元に完全に再現し、全252件の既存テストおよびE2Eテストをパスさせました。
