# タスク: Langiumを用いたフルパーサー化

## 概要
SPD (Simple PAD Description) フォーマットのパーサーにおいて、既存の「1行パース + 手動でツリー構築」のアプローチ（アプローチA）から、「Langiumのフル機能を用いたインデントベースのファイル全体パース」（アプローチB）へ移行する。

## 目標
1. `spd.langium` の拡張: インデントベースのブロック構造とマルチラインの正規表現をサポートし、ファイル全体のASTを一度に構築できるようにする。
2. カスタム TokenBuilder / Lexer の実装: インデントの増減に応じた `INDENT`, `DEDENT` トークンの生成を行う。
3. `parser.ts` の刷新: 手動での状態管理やタブカウントを廃止し、LangiumのASTから既存の `Node` クラスへの変換処理のみにリファクタリングする。
4. テストの全通過: 既存のテストがすべてパスすることを確認する。
